# Manus（旧manul）からWordPressへの自動投稿 設定ガイド

Manusの定期タスクで生成した記事と画像を、連携アプリ「Make」を使ってWordPressに完全自動で投稿する具体的な設定手順です。

---

## 準備編：WordPress側の設定（連携プラグインの導入）

Makeの最新のWordPressモジュールでは、直接のパスワード入力ではなく、専用プラグインを使った安全な「API Key」方式が採用されています。

1. **WordPressの管理画面** にログインします。
2. 左のメニューから **「プラグイン」 ＞プラグインを「新規追加」** を開きます。
3. プラグインの検索窓で **「Make」** と検索します。
   （「Make」という紫色のロゴのプラグインが見つかります）
4. **「今すぐインストール」** を押し、完了したら **「有効化」** します。
5. 有効化すると、WordPressの左メニュー（設定の下など）に **「Make」** という項目が表示されるので開きます。
6. 設定画面の中に **API Key（APIキー）** が表示されています。
   > [!IMPORTANT]
   > このAPI Keyをコピーしておいてください。（このあとのMake側の接続画面で貼り付けます）

## 設定編：Make（旧Integromat）の設定

Makeのアカウント（無料）を作成し、自動化の「シナリオ（流れ）」を作ります。

### ステップ1：Makeアカウントの作成と新しいシナリオの作成
1. [Make公式サイト](https://www.make.com/en) にアクセスし、無料でアカウントを作成します。
2. ログイン後、左側のメニューから **「Scenarios（シナリオ）」** を選び、右上の **「+ Create a new scenario」** をクリックします。

### ステップ2：Manusからデータを受け取る「窓口」を作る
1. 画面中央の大きな「＋」ボタンをクリックし、検索窓に **「Webhooks」** と入力して選択します。
2. リストの中から **「Custom webhook」** を選びます。
3. 「Add」ボタンを押し、Webhookの名前（例: `Manus WP Webhook`）をつけて **「Save」** を押します。
4. `https://hook.usX.make.com/...` のような **URL（Webhookアドレス）** が生成されるので、これを「Copy address to clipboard」ボタンでコピーします。
   > [!NOTE]
   > このWebhook URLは、後ほどManusのプロンプトに組み込んで使用します。
5. ポップアップの「OK」を押します。

### ステップ2.5：画像のダウンロード（HTTPモジュール）
Manusからは「画像データそのもの」ではなく「画像のURL（`image_url`）」が送られてきています。そのため、アップロードする前にURLから画像をダウンロードする設定を挟みます。
1. Webhookの右側にモジュールを追加し、検索窓で **「HTTP」** を探して選びます。
2. アクションの中から **「Get a file」** を選択します。
3. 設定画面の **「URL」** の欄をクリックし、リストの中からWebhookから届いている `image_url` を選んで「OK」を押します。

### ステップ3：WordPressに画像をアップロードする
1. さらに右側にモジュールを追加（Add another module）します。
2. 検索窓に **「WordPress」** と入力して選択し、Media Itemsという項目の中にある **「Create a Media Item」** を選びます。（これがファイルのアップロード機能に該当します）
3. **Connection（接続）部分の「Add」を押します：**
   - **Wordpress REST API base url:** `https://crypto-shift.com/wp-json/`
     （※画面の例にあるように、末尾に `/wp-json/` を加えてください）
   - **API Key:** 今回【準備編】で発行した **MakeプラグインのAPI Key** を貼り付けます。
   これを入力して Save を押します。
4. **File 項目の設定（ここが重要です）:**
   - 画像をアップロードするために、先ほどの「HTTP」で取得したデータを紐づけます。
   - 「File」という項目の中で、ファイルの参照元として「HTTP - Get a file」を選びます。
   - もし手動でマッピング（Map）する画面になっている場合は、以下のように設定します。
     - **File name:** `thumbnail.jpg` のように適当な名前を入力します（※固定でも動きますが、 `title` を使って `{{title}}.jpg` のように設定すると管理しやすくなります）。
     - **Data:** 「HTTP - Get a file」から出力されている `Data` を選択します。
   設定ができたら「OK」で保存します。

### ステップ4：WordPressで記事を作成する
1. 画像アップロードモジュールの右側に、さらにモジュールを追加します。
2. 再び **「WordPress」** を選び、今度は **「Create a Post」** を選びます。
3. 記事の設定枠が開きますので、以下のように紐づけます：
   - **Title:** Manusから送られてきたタイトルを指定
   - **Content:** Manusから送られてきたHTMLを指定
   - **Status:** 最初は安全のため `Draft`（下書き）に設定しておくことをお勧めします。
   - **Featured Media (アイキャッチ画像):** ここで、1つ前のステップ（画像をアップロード）から出力される **「Media ID (Attachment ID)」** を紐づけます。これにより、記事のアイキャッチが自動設定されます。
4. 設定を保存し、シナリオ画面左下の **「Save」** アイコン（フロッピーディスクのマーク）をクリックします。

## 仕上げ編：Manusのプロンプト変更

Manusのタスクが完了時にMake（Webhook）へデータを送信するよう、現在のプロンプトを以下のように修正します。

プロンプトの最後にあるセクション部分を、以下のように書き換えてください。

```markdown
## 8. Final Output Execution
1. **記事の生成:** 上記のルールに従い、高品質な分析記事を執筆し、HTML形式のテキストデータとして出力する。
2. **画像の生成:** 記事の内容にマッチしたプロフェッショナルな画像を生成する。
3. **Makeへの自動データ送信（重要）:** 
   Manusのコネクタ（APIまたはWebhook連携機能）を使用し、この記事の「タイトル」、「生成したHTMLテキスト（本文）」、および「生成した画像データ（または画像のURL）」を以下のWebhook URLへ確実に送信（POST）してください。
   【Webhook URL】: （※ここにMakeでコピーしたWebhook URLを貼り付けます）
4. **完了:** データ送信の成功レポートをもってタスク完了とする。

## 9. Technical Output Syntax
WordPressへの貼り付けを想定し、出力内容は必ず以下の形式を厳守してください。
- **HTML形式での出力:** Markdown記法（# や * など）は一切使用せず、そのまま貼り付け可能な「HTMLタグ形式」で本文を出力してください。
- **使用タグの例:** `<h2>`, `<h3>`, `<p>`, `<ul>`, `<li>`, `<strong>`, `<a>` 等
```

### 最終テストの流れ
1. Makeのシナリオ画面左下にある **「Run once（1回だけ実行する）」** ボタンを押して、待機状態にします。
2. Manusで定期タスク（または手動でのテスト実行）を動かします。
3. Manusが記事を作り終え、Makeにデータを送信すると、Makeの画面上で各アイコンに緑色のチェックマークが付き、データが流れるように動きます。
4. ご自身のWordPress（crypto-shift.com）の管理画面にログインし、対象の記事が「下書き」として入り、アイキャッチ画像がきれいに設定されていれば成功です！
5. 無事にテストが成功したら、Makeシナリオ画面左下にあるトグルスイッチを **「ON（Scheduling）」** に切り替えて、全自動化を有効にします。
